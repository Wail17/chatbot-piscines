# Fix Summary: FAQ Matching and GPT Fallback

## Problems Fixed

### Problem 1: FAQ Questions with Empty Answers
**Issue**: FAQ questions that exist in FAQAI.jsonl but have empty or very short answers were returning unhelpful responses or error messages.

**Examples**:
- "Wat moet ik doen om mijn wifipool apparaat te resetten?" - had empty answer
- "Waar kan ik het serienummer van mijn wifipool apparaat vinden?" - had empty answer
- "Mijn wifipool gaat steeds offline" - had short unhelpful answer "Kies hieronder je toesteltype"

**Root Cause**: The system was correctly finding FAQ entries but returning empty/minimal answers without falling back to GPT.

**Fix**: Modified `_respond_for_row()` function in `app/main.py` to:
1. Check if FAQ answer is empty or shorter than 30 characters
2. Automatically fall back to GPT for intelligent answer generation
3. Add a note that the answer is AI-generated
4. Provide helpful fallback message if GPT is unavailable

### Problem 2: GPT Fallback Not Working for Non-FAQ Questions
**Issue**: Questions not in FAQ should use GPT fallback but weren't working properly.

**Examples**:
- "Waarom is mijn water groen?"
- "Wat is de beste temperatuur voor zwembad?"
- "Hoeveel chloor moet ik gebruiken?"

**Fix**: The GPT fallback was already implemented at line 2547 but was not being called when FAQ had empty answers. Now it's called in both cases:
1. When no FAQ match is found (existing behavior)
2. When FAQ match is found but answer is empty/too short (new behavior)

### Problem 3: Missing Source Field
**Issue**: No way to distinguish between FAQ answers and AI-generated answers.

**Fix**: Added `"source"` field to ALL response types:
- `"source": "faq"` - Answer from FAQ database
- `"source": "ai_fallback"` - Answer generated by GPT
- `"source": "correction"` - Answer from admin corrections
- `"source": "rag"` - Answer from RAG retrieval

## Configuration Required

For GPT fallback to work, ensure `OPENAI_API_KEY` environment variable is set:
```bash
export OPENAI_API_KEY="your-api-key-here"
```

Without this, the system will fall back to a friendly error message asking users to contact support.
